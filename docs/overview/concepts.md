# 核心概念

## 上下文管理（Context）

Agent 的"工作记忆"——它在执行任务时能看到什么信息。

| 类型 | 说明 | 类比 |
|------|------|------|
| **系统提示** | 定义 Agent 的角色和行为规则 | "你是一个客服助手，用中文回答" |
| **对话历史** | 用户和 Agent 之间的消息记录 | 聊天记录 |
| **工具返回值** | 工具调用后的结果 | 查完天气后返回的数据 |
| **检索结果** | 从知识库中找到的相关文档 | 图书管理员帮你找到的资料 |
| **外部注入** | 手动传入的额外信息 | "这是客户的订单号" |

> 上下文太多会超出模型的上下文窗口，需要**压缩、摘要、裁剪**——这就是上下文工程要解决的问题。

## 记忆体系（Memory）

| 类型 | 时长 | 存储位置 | 用途 |
|------|------|----------|------|
| **短期记忆** | 单次会话 | 内存中 | 当前对话的上下文 |
| **长期记忆** | 跨会话 | 数据库/文件 | 用户偏好、历史总结、个人资料 |
